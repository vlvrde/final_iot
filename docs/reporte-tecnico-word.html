Reporte Técnico del Sistema IoT con MQTT

Proyecto: Sistema de monitoreo y control IoT mediante MQTT

Equipo: Equipo 7

Broker: BeagleBone Black con Mosquitto

Publisher: ESP32 con 7 sensores reed switch

Subscriber: Dashboard web en Python

Fecha de validación: 20 de junio de 2026

1. Introducción

En este proyecto se construyó un sistema IoT para vigilar la apertura de siete compartimentos. El sistema usa el protocolo MQTT para que cada parte cumpla una tarea clara. El ESP32 toma los datos de los sensores, la BeagleBone Black recibe y reparte los mensajes, y el dashboard web muestra el estado en tiempo real y activa una alarma digital cuando detecta aperturas.

2. Objetivo

Diseñar e implementar una arquitectura IoT simple, ordenada y funcional, donde un nodo publicador, un broker MQTT y un nodo suscriptor trabajen juntos en tiempo real. El sistema debía permitir lectura de sensores, envío de eventos, visualización web y reacción del actuador digital.

3. Descripción general del sistema

La solución final quedó dividida en tres partes:

  • ESP32: lee siete sensores reed switch y publica el estado de cada compartimento.
  • BeagleBone Black: ejecuta el broker Mosquitto y escucha en el puerto 1883.
  • Dashboard web: se suscribe a los tópicos MQTT, muestra el estado y activa una alarma digital.

9. Desarrollo e implementación

9.1 Publisher

El ESP32 se programó en C++ con PlatformIO y el framework Arduino. El nodo lee siete sensores reed switch, detecta cambios de apertura y cierre, y publica tres tipos de mensajes: estado por compartimento, resumen general y heartbeat.

9.2 Broker

El broker se ejecutó en la BeagleBone Black con Mosquitto. Durante la configuración se detectó que la imagen Debian 7 de la BBB era antigua, por lo que fue necesario ajustar la compatibilidad del proyecto para trabajar con MQTT 3.1.

9.3 Subscriber

El subscriber se desarrolló como una aplicación web en Python. La interfaz muestra el número de compartimentos abiertos, el estado del publisher, la alarma activa y una lista de eventos recientes.

10. Pruebas de conectividad y análisis de resultados

Se hicieron pruebas de red, de broker y de intercambio MQTT entre los nodos. Las pruebas ayudaron a detectar un problema de compatibilidad con la versión del broker de la BBB. Después del ajuste, la comunicación quedó estable.

Prueba Comando o evidencia Resultado esperado Resultado observado
Broker activo netstat -ltnp | grep :1883 Mosquitto escucha en el puerto 1883 Correcto. La BBB mostró escucha en 0.0.0.0:1883 y :::1883.
Conectividad TCP desde laptop Prueba de socket al puerto 1883 Conexión TCP exitosa Correcto. La laptop abrió conexión al broker en 192.168.1.85:1883.

10.1 Hallazgo técnico importante

Durante las pruebas se encontró que el broker viejo de la BeagleBone Black aceptaba MQTT 3.1, pero no trabajaba bien con clientes configurados en MQTT 3.1.1. Esto causó que el ESP32 lograra abrir el socket TCP, pero se quedara esperando la respuesta MQTT. La solución fue compilar el cliente del ESP32 y el dashboard con protocolo MQTT 3.1.

10.2 Estado observado en la validación

  • El broker Mosquitto quedó activo en la BBB.
  • El ESP32 se conectó por WiFi y publicó datos al broker.
  • El monitor MQTT recibió eventos y heartbeats en tiempo real.
  • El dashboard mostró 7 compartimentos abiertos durante la captura final.

12. Conclusiones

El proyecto cumplió con la arquitectura pedida en la asignación. Se implementaron de forma correcta los tres roles principales del sistema: publisher, broker y subscriber. El ESP32 envía datos en tiempo real, la BeagleBone Black distribuye los mensajes con Mosquitto y el dashboard reacciona de forma inmediata.

También se comprobó que un problema de compatibilidad de versiones puede afectar la comunicación, aun cuando la red y el puerto estén bien. Al corregir el uso de MQTT 3.1, el sistema quedó funcional. Con esto se demuestra que la solución no sólo funciona, sino que además fue depurada y validada en laboratorio.
